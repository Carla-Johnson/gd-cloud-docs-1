Deploy Rails app with Puma and NGINX – Arch Linux

Difficulty: 2
Time: 30 minutes

Puma is a simple, fast and highly concurrent HTTP 1.1 server for Ruby web applications. It is designed for running Rack apps. This article will help you to deploy a Rails application with default database PostgreSQL, using Puma and Nginx on Arch Linux .

Prerequisites

Please refer the following links to prepare your deployment environment:

  1. 17522 - Install Ruby on Rails with rbenv - Arch Linux
  2. 17527 - Use PostgreSQL with your Ruby on Rails Application - Arch Linux

Create a Rails application

  1. Create a Rails application.
    rails new myapp -d postgresql
  2. Move to the application directory.
    cd myapp

Create a database user for production

  1. When creating a database user, we suggest that you keep the production username same as your application name so it can be easier to use and remember.
    sudo -u postgres createuser -s myapp
  2. Provide a password to the database user.
    sudo -u postgres psql
  3. Type the following command in the PostgreSQL prompt:
    \password myapp
  4. When prompted for the password, enter a desired password and exit.
    \q
  5. Make sure that you are in Rails application’s root directory. If not, then change the directory to Rails application directory.
    cd /path/to/myapp
  6. Open the database configuration file i.e. database.yml.
    sudo vim config/database.yml
  7. Find pool: 5 in the default section of database.yml file. Just below the pool: 5 line, add database connection string to connect. 
    pool: 5
host: localhost
  8. Move to the production section where you will find the following lines: 
    production:
  <<: *default
  database: myapp_production
  username: myapp
  password: <%= ENV['MYAPP_DATABASE_PASSWORD'] %>

<<: *default this line will add configuration from default section
database section , type your database name here

    In the username section, check if the user is similar to what you have created earlier. If not, then change it as per your requirement.

Install Pluggin for rbenv-vars

You should set a secret key and database password by using environment variable, before you deploy Rails application in production. Here we are describing a way to manage environment variables by using rbenv-vars plugin that loads password and secret key at the runtime. Next, we are going to install and configure rbenv-vars for our application.

  1. Setup the rbenv-vars by changing directory to .rbenv/plugins using the following command and clone the repository of rbenv-var from github
    cd ~/.rbenv/plugins
    git clone https://github.com/sstephenson/rbenv-vars.git

    Note: Ensure that you’ve configured / install rbenv in your machine correctly.

  2. Set environment variable by changing your directory to application directory and generate the secret key.
    cd ~/myapp
    rake secret
  3. Once you have generated the secret key, you will find a token key. Copy that key and replace. To do this, open .rbenv-vars file into your favorite editor and replace your key with your_generated_secret key.
    vim .rbenv-vars
    SECRET_KEY_BASE=your_generated_secret

    Environment variable that you set in this file will be read by your Rails application.

  4. Set the database password for connection.
    MYAPP_DATABASE_PASSWORD=production_database_password
  5. Set your database environment variable similar to the one that is defined in the database configuration file i.e database.yml.
  6. Save and Exit.
  7. Check the environment variables list that you have set in file by using the following command:
    rbenv vars

Setup production database

  1. Once your application is able to communicate with the PostgreSQL database, you can create the production database.
    RAILS_ENV=production rake db:create
  2. Generate the controller and database migration using the following command:
    rails generate scaffold Task title:string note:text
  3. Migrate to the production database using the following command:
    RAILS_ENV=production rake db:migrate
  4. Precompile the assets.
    RAILS_ENV=production rake assets:precompile
  5. Test the application environments, where you can bind your public IP too.
    RAILS_ENV=production rails server --binding=server_public_IP
  6. Now open the browser and type below mentioned in the address bar. http://server_public_IP:3000/tasks

Install Puma

  1. Go back to your Rails application server and terminate it by using ctrl+c.
  2. Open the Gemfile.
    vim Gemfile
  3. Add the following lines at the end of the file. Before you proceed, make sure that you are in the application’s root directory.
    gem 'puma'
    gem 'foreman'
  4. Save and exit from the editor.
  5. To install Puma and its dependencies, use the bundler command.
    bundle install
  6. This will take some time for installation.

Configure Puma

  1. Check the CPU cores before you proceed to configure the Puma server. This will help us to set workers and other process assignments.
    grep -c processor /proc/cpuinfo
  2. Open the Puma configuration file into any of your favorite text editor.
    vim config/puma.rb
  3. Copy and paste below configuration in the Puma file. 
    #!/usr/bin/env puma

# app do |env|
#   puts env
#
#   body = 'Hello, World!'
#
#   [200, { 'Content-Type' => 'text/plain', 'Content-Length' => body.length.to_s }, [body]]
# end

environment 'production'
daemonize false

pidfile '/home/imuser/myapp/tmp/pids/puma.pid'
state_path '/home/imuser/myapp/tmp/pids/puma.state'

# stdout_redirect 'log/puma.log', 'log/puma_err.log'

# quiet
threads 0, 16
bind 'unix:///home/imuser/myapp/tmp/sockets/puma.sock'

# ssl_bind '127.0.0.1', '9292', { key: path_to_key, cert: path_to_cert }

# on_restart do
#   puts 'On restart...'
# end

# restart_command '/u/app/lolcat/bin/restart_puma'


# === Cluster mode ===

workers 2 #change workers as per your cpu core


# on_worker_boot do
#   puts 'On worker boot...'
# end

# === Puma control rack application ===

activate_control_app 'unix:///home/imuser/myapp/tmp/sockets/pumactl.sock'
  4. Once you have pasted the configuration lines, change the workers as per your CPU cores. Save your changes and exit.

Start, Stop and Restart commands for Puma

Below are the commands to start, stop and restart the Puma services.

  1. Move to the application directory.
    cd ~/myapp
    mkdir tmp/sockets tmp/pids
  2. To start the Puma services, type: 
    bundle exec puma -C config/puma.rb –d --pidfile = tmp/pids/puma.pid
-d is for Daemonizing...
  3. To stop the Puma services, type
    kill –s SIGTERM `cat tmp/pids/puma.pid`
  4. To restart the Puma services, type:
    kill –s SIGUSR2 `cat tmp/pids/puma.pid`

Install and setup Nginx

  1. Install Nginx using pacman.
    sudo pacman install nginx
  2. Now open the default server block with a text editor. 
    cd /etc/nginx/
sudo vim nginx.conf
  3. Find the line ‘HTTPS server’ in the file and replace the below configurations in the configuration file. 
    upstream myapp {
server unix:///home/imuser/myapp/tmp/sockets/puma.sock fail_timeout=0;
}
server {
  #listen 80 deferred;
  server_name coolexample.com www.coolexample.com;
  root /home/imuser/myapp/public;
  try_files $uri/index.html $uri @myapp;
  location @myapp {
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header Host $http_host;
    proxy_redirect off;
    proxy_pass http://myapp;
  }
  error_log /home/imuser/myapp/log/error.log;
  error_page 500 504 /500.html;
  error_page 502 /502.html;
  error_page 503 /503.html;
  client_max_body_size 4G;
  keepalive_timeout 10;
}
  4. Save and exit. This configures Nginx as a reverse proxy, so HTTP requests get forwarded to the Puma application server via a UNIX socket. Feel free to make any changes as required.
  5. Restart Nginx service to put the changes into effect.
    sudo systemctl restart nginx
  6. You can now access the production environment of your Rails application using the public IP address or FQDN of the server. Visit the application server in a web browser to access the tasks controller.
    http://server_public_IP/tasks
