Common Linux log file locations - ALL DISTROS

Difficulty: 1
Time: 15 minutes

Knowing the locations where you can find logs in the Linux environment can be quite useful at times. If you encounter problems with your Linux environment, these locations would be a good place to start looking for logs to help you indentify what might be causing the issue.

  • /var/log – Location where logs files are usually placed.
  • /var/log/alternatives.log – Tracks messages from the update-alternatives command.
  • /var/log/anaconda.log – Tracks messages related to Linux installation and provisioning. This is useful when installing a new Linux system.
  • /var/log/audit – Location where auditd, the audit daemon, writes its logs.
  • /var/log/auth.log – Tracks usage of authorization systems. This log is useful for learning about user logins and usage of the sudo command.
  • /var/log/boot.log – Log containing system bootup information.
  • /var/log/btmp – Tracks messages related to unsuccessful login attempts. Use the lastcommand to view the content of this log.
  • /var/log/conman – This directory is the location where the ConMan client writes its logs.
  • /var/log/cron – Tracks messages from the cron daemon. The daemon logs information about jobs as it starts them.
  • /var/log/cups – Tracks the messages related to the printer and printing.
  • /var/log/daemon.log – Tracks messages from the various daemons that running on the system.
  • /var/log/dmesg – Contains system bootup information. The system writes boot messages to this log. You can view the full kernel ring buffer messages with the dmesg command.
  • /var/log/dpkg.log – Tracks messages about installations that use dpkg to install or remove packages.
  • /var/log/faillog – Tracks failed user login attempts. Use the faillog command to view the contents of this log.
  • /var/log/httpd and /var/log/apache2 – Location where the Apache web server writes logs, such as error_log and access_log.
  • /var/log/kern.log – Tracks messages from the kernel. This is useful when trying to troubleshoot a custom built kernel.
  • /var/log/lastlog – Tracks recent user login information. The lastlog command should used to view the content of this log.
  • /var/log/lighttpd – Directory where the light HTTPD error_log and access_log logs are written to.
  • /var/log/mail – Directory where additional logs from your mail server can be found. Sendmail stores a statistics file along with other files at this location.
  • /var/log/maillog and /var/log/mail.log – These logs track messages from the mail server running on the system.
  • /var/log/messages – This log is a global system log that is used by various systems such as mail, cron, daemon, kern, and auth. It also includes the messages that are logged during system startup.
  • /var/log/prelink – Directory where prelink writes details about .so files that it has modified.
  • /var/log/sa – Directory where sysstat package writes its daily sar files.
  • /var/log/samba – Directory where samba, which connects Windows to Linux, writes its logs.
  • /var/log/secure – Tracks information related to authentication and authorization privileges. Sshd logs messages here about unsuccessful login attempts.
  • /var/log/setroubleshoot – Directory where setroubleshootd, the SELinux Trouble Shoot Daemon, writes information about a files security context.
  • /var/log/sssd – Directory where the system security services daemon writes information about remote access and authentication.
  • /var/log/user.log – Tracks messages about all user level logs.
  • /var/log/wtmp or /var/log/utmp – Log containing the login records. Use wtmp to find out who is logged into the system. The who command displays information contained in this log.
  • /var/log/Xorg.x.log – Tracks messages from the XWindows system.
  • /var/log/yum.log – Tracks messages about installations that use yum to install or remove packages.
