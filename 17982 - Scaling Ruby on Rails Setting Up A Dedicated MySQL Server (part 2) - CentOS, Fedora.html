Setting up a dedicated MySQL server to host Ruby on Rails application servers, Ruby, Rails, MariaDB, MySQL, NoSQL, Scaling

Scaling Ruby on Rails: Setting Up A Dedicated MySQL Server (part 2) - CentOS, Fedora

Difficulty: 4

Time: 1 hour

In this article, we will be creating a database layer using MySQL to host the application servers running Ruby on Rails. This articles will assist you in setting up a dedicated MYSQL server and scaling Ruby on Rails applications between the servers in CentOS or Fedora Linux distributions.

Note: We assume that you are logged in as a sudo user and installed Ruby on Rails using RVM in CentOS

Step 1: Select the Database

Based on the needs and application type, choose your database server to build a database layer for your application development. Following are some popular databases:
  • For Relational database server: Use MySQL and derivatives, PostgreSQL
  • For Non-relational database servers: Use Column-based Cassandra and HBase, Document-based MongoDB and Couchbase and Graph-based OrientDB, Neo4J and related databases

Step 3: Add the Database Server to the Deployment Set-Up

  1. Let's create and configure MySQL database. To prepare the setup, type
    sudo yum -y update
  2. To install the development tools, run the following command
    sudo yum groupinstall -y 'development tools'
  3. To use the YUM package, add the Extra Packages for Enterprise Linux (or EPEL) software repository
    sudo wget http://dl.fedoraproject.org/pub/epel/7/x86_64/e/epel-release-7-5.noarch.rpm
    sudo rpm -ivh epel-release-7-5.noarch.rpm
  4. Again update everything and install the required libraries
    sudo yum -y update
    sudo yum install -y curl-devel vim sqlite-devel libyaml-devel

Step 4: MySQL Installation

  1. Download and install the MySQL database server
    sudo yum install mysql-server mysql-devel
    If MySQL server doesn't work, then try MariaDB
    sudo yum install MariaDB-server MariaDB-client
  2. Now initiate the MySQL server daemon by typing
    sudo service mysqld start

    Or

    sudo /etc/init.d/mysql start
  3. Run the MySQL installation process, by typing
    sudo/usr/bin/mysql_secure_installation
  4. After you have finished the above command, a welcome screen displays the following 
      NOTE: RUNNING ALL PARTS OF THIS SCRIPT IS RECOMMENDED FOR ALL MySQL
          SERVERS IN PRODUCTION USE!  PLEASE READ EACH STEP CAREFULLY!


  In order to log into MySQL to secure it, we'll need the current 
      password for the root user.  If you've just installed MySQL, and 
        you haven't set the root password yet, the password will be blank, 
        so you should just press enter here.
  
  Enter current password for root (enter for none):

    If you have created a password by using the following command, then hit Enter

       /usr/bin/mysqladmin -u root password 'new-password'
   /usr/bin/mysqladmin -u root -h myt password 'new-password'

    Next, answer the following questions

        Set root password?                     --> Y
    Remove anonymous users?                --> Y
    Disallow root login remotely?          --> Y
    Remove test database and access to it? --> Y
    Reload privilege tables now?           --> Y
  5. Establish database connection using MySQL client and enter the password for root that was previously step-up
    sudo mysql -u root -p
  6. Make a new database
    create database rails_myapp;
  7. Create new users for your database and grant remote and local access to this user
    CREATE USER 'rails_myapp_user'@'localhost' IDENTIFIED BY 'pwd';
    CREATE USER 'rails_myapp_user'@'%' IDENTIFIED BY 'pwd';

    Type the following command to confirm that users are created

    SELECT User,host FROM mysql.user;
  8. Grant access rights to any specific user by typing
    GRANT ALL ON rails_myapp.* TO 'rails_myapp_user'@'localhost';
    GRANT ALL ON rails_myapp.* TO 'rails_myapp_user'@'%';

    And then revoke the privileges

    FLUSH PRIVILEGES;

    Exit from Database

    Exit
  9. To enable remote connections, open the MySQL configuration my.cnf and edit using the vim text editor
    sudo vim /etc/my.cnf

    And type the below mentioned line in this file

    [mysqld]
..
.
bind-address   =  0.0.0.0

Step 5: Setup your Rails application

  1. Install gem for Mysql and update
    sudo gem install mysql2
    sudo gem update -system
  2. Install Rails package with following command
    sudo gem install rails
  3. Then create a Rails application and Use the -d mysql option to set MySQL as the database
    rails new Appname -d mysql
  4. Next, move to the application directory
    cd Appname

  5. You'll find the database configuration settings for your Rails applications within the database.yml file in /config directory

  6. Modify the database.yml file with the help vim text editor, by typing
    sudo vim config/database.yml
  7. Swap the production: YML section with the below mentioned lines by editing the contents with the your own required configuration 
    production:
  adapter: mysql
  encoding: utf8
  database: rails_myapp
  username: rails_myapp_user
  password: pwd
  host: 128.199.233.36
  port: 3306
  pool: 10

    Save and close

  8. Open the Gemfile
    sudo vim Gemfile

    And append the below mentioned line

    gem 'mysql'

    Save and close

  9. Setup gem using bundle
    bundle install
